# OlaForge - 智能安全沙箱系统架构设计

## 一、项目背景

本项目整合 SkillLite 和 CubeSandbox 的优点，创建一个更完美的 AI Agent 安全沙箱系统。

### 基础参考

| 项目 | 优点 | 缺点 |
|------|------|------|
| SkillLite | 自进化引擎、轻量级、零依赖 | 硬件隔离弱、集群能力弱 |
| CubeSandbox | 硬件级隔离、高性能、企业级 | 无智能化、部署复杂、运维要求高 |

---

## 九、安全分级设计

针对评审中提出的安全分级需求，设计四级安全模式：

### 9.1 安全等级对比

| 等级 | 名称 | 隔离方式 | 适用场景 | 安全评分 |
|------|------|----------|----------|----------|
| L0 | 快速模式 | 无隔离 | 可信代码、快速测试 | 0/20 |
| L1 | 轻量级 | namespace + caps | 个人开发、无第三方 | 12/20 |
| L2 | 标准级 | bwrap/seccomp | 一般使用、外部代码 | 16/20 |
| L3 | 增强级 | KVM MicroVM | 敏感场景、高安全 | 20/20 |

### 9.3 自动选择策略

```python
# 自动安全等级选择
def auto_security_level(code_source: str, has_network: bool) -> str:
    if is_trusted_code(code_source):
        return "L0"  # 快速模式
    elif has_network and not is_verified():
        return "L3"  # 增强级
    else:
        return "L2"  # 默认标准级
```

## 十三、架构评审结论

### 与 SkillLite 对比

| 维度 | SkillLite | OlaForge | 优势 |
|------|-----------|----------|------|
| 本地易用 | 好 | 更好 | 分级设计 |
| 安全性 | 好 | 更好 | 多层防护+KVM |
| 智能化 | 好 | 更好 | 完善进化流程 |
| 企业支持 | 弱 | 强 | K8s 部署 |

### 与 CubeSandbox 对比

| 维度 | CubeSandbox | OlaForge | 优势 |
|------|-------------|----------|------|
| 本地易用 | 差 | 优 | 零依赖 |
| 智能化 | 无 | 有 | 进化引擎 |
| 部署复杂度 | 高 | 低 | 渐进式 |
| 成本 | 高 | 低 | 本地免费 |

### 综合评价

✅ **优势明显**:
1. 本地易用性远超两者
2. 安全性继承并增强
3. 智能化填补 CubeSandbox 空白
4. 部署复杂度大幅降低

⚠️ **仍需验证**:
1. MVP 实际运行效果
2. 性能指标是否达标
3. 用户体验是否流畅
